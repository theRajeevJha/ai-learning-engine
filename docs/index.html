How It Works

Student Answer
      ↓
Mastery Engine updates probability
      ↓
Knowledge Graph validates prerequisites
      ↓
Curriculum Planner selects next concept (ZPD)
      ↓
Recommendation Engine returns next exercise
      ↓
Repeat (adaptive loop)

API Documentation

Base URL

https://api.yourdomain.com/v1

Endpoints

POST /placement/start
→ Returns student ability estimate

POST /answer/submit
→ Updates mastery state

GET /recommendation/{studentId}
→ Returns next concept + exercise

GET /student/{id}/profile
→ Returns full mastery map

Example Response

{
  "nextConcept": "loops",
  "exercise": "loop_ex_3",
  "difficulty": 0.62
}

System Evolution

Phase 1 (Current)

Deterministic adaptive learning engine (graphs + rules)

Phase 2

LangGraph agents: feedback, tutoring, misconception detection

Phase 3

AI-driven curriculum optimization + question generation

Problem

Traditional learning systems follow a fixed syllabus for all students, ignoring individual knowledge gaps and learning speed.

  • No personalization
  • No concept dependency awareness
  • No adaptive difficulty
  • No real-time mastery tracking
